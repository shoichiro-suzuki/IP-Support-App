# Doc Style Guard

## Overview
- DDDで作る仕様/メモの構成・粒度・文体を揃えるチェックリスト。
- トリガー: docs/*.md, AGENTS.md, CLAUDE.md 追加・更新時。

## Quick start
- 対象を決める: 仕様/運用/タスク/TODOどれか。
- テンプレ参照: `references/templates.md` の該当節を使用。
- 記法: 箇条書き・短文、必要なら `- [ ]` でタスク表現。
- 終了チェック: 下記チェックリストで確認。

## チェックリスト（共通）
- 目的とスコープが先頭にあるか。
- 入出力/依存/制約が箇条書きで1行ずつ。
- 変更理由や決定事項を短文で残したか。
- 専有ルール遵守（簡潔性、絵文字禁止、参照パス明示）。
- TODOは `- [ ]` / `[x]`、命令形。
- ファイル参照はパス単体で記載。

## 文書タイプ別指針
- 仕様（API/機能）: 目的、前提、入出力、主要フロー、エラー、TODO、テスト観点。1行1項目。
- UIメモ: 画面目的、主要要素、イベント/遷移、状態、バリデーション、未決事項。
- 運用/設定: 対象環境、前提、コマンド、ENV/ファイル、ロールバック、既知課題。
- メモ/決定記録: 背景、決定、理由、影響、TODO。

## 手順
1. `list_changes.sh` や `git status` で対象ドキュメントを決定。
2. テンプレを基に不足項目を埋める（不要項目は削除）。
3. 記法/粒度チェック（上記チェックリスト）。
4. 関連TODOやテスト観点を追加。

## references/
- `references/templates.md`: 文書タイプ別テンプレ（機能計画/技術スタック/データスキーマ/アーキテクチャ/API仕様/UI計画・モック/TODOS/テスト/運用/決定記録）。必要に応じてコピペし不要項目を削る。

# Readme Author

## 目的
- READMEの目的/範囲を短く示し、初見の開発者/利用者の導線を作る
- 日本語のみ、目次は大項目のみ

## 参照ソース
- 既存README（あれば最優先）
- `docs/`（概要/運用/仕様）
- `requirements*.txt`（主要ライブラリ）
- `configs/`（設定/スキーマ）
- `Home.py`, `pages/`, `services/`, `api/`（主要機能）

## ワークフロー
1. 参照ソースを確認し、概要/機能/設定/起動/テストを抽出
2. 不足情報を質問（環境変数、起動コマンド、運用/デプロイ）
3. 目次を大項目のみで提示
4. 利用者向け・開発者向けの大項目で本文を作成
5. 冗長さを削り、箇条書き中心で短文化

## READMEの大項目（例）
- 概要
- 利用者向け
- 開発者向け
- 設定/環境変数
- 開発/テスト
- 構成
- 付録（FAQ/ライセンス/問い合わせ）

## 参照ファイル
- `references/readme_sections.md` を読む: 章構成と質問リストのチェック

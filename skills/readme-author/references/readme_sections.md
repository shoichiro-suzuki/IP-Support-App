# READMEセクションガイド

## 章構成の指針
- 大項目の目次のみ（細かい小見出しは本文内で最小限）
- 利用者向け/開発者向けを分ける
- 手順は箇条書きで短く

## 章ごとの要点

### 概要
- 何のアプリか（1-2文）
- 主要機能（3-5点）

### 利用者向け
- できること
- 操作フロー（最短手順）
- 出力物/制約

### 開発者向け
- 動作環境（OS/言語/主要依存）
- セットアップ/起動
- 主要ディレクトリ

### 設定/環境変数
- 必須ENVのみ列挙
- 用途を短く添える

### 開発/テスト
- 実行コマンド
- テスト方針（あれば）

### 構成
- 重要ディレクトリのみ

### 付録
- FAQ/ライセンス/問い合わせ（必要時のみ）

## 追加で確認する質問
- 起動コマンドとポート/URL
- .env/設定ファイルの場所
- デプロイ/運用手順の有無

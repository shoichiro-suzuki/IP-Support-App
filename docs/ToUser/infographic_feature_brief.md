# インフォグラフィック原稿（機能説明）

### 0. 1行説明（キャッチ）

- アプリ名：契約審査サポートアプリ
- 目的（誰の、何を楽にする？）：契約書のアップロード〜条文抽出〜審査結果確認を一気通貫
- 想定利用者：法務/契約審査担当、事業部担当、管理者

### 1. 利用者ができること（全体像：3〜6個に圧縮）

1. 契約書をアップロードして条文抽出・審査を実行する
2. 条項ごとの懸念点と紐付くナレッジを確認する
3. 審査結果をCSVで出力する
4. 審査チャットで追加確認する
5. ナレッジを検索・閲覧する（管理者は編集も可能）

### 2. 代表ユーザーフロー（3本まで）

**フローA：新規審査〜結果出力**

1. 起動
2. 契約審査ページを開く
3. `.docx/.pdf` をアップロード
4. 条文抽出
5. ナレッジ照合/LLM審査
6. 結果確認（懸念条項）
7. CSVダウンロード

**フローB：追加確認（審査チャット）**

1. 結果画面でチャットを開く
2. 条項/懸念点を質問
3. 追加回答を確認

**フローC：ナレッジ検索/更新**

1. ナレッジ管理を開く
2. 契約種別/キーワードで検索
3. 詳細を確認
4. （管理者）編集/保存

### 3. 機能一覧（カード化前提）

#### 3.1 コア機能（最重要：5〜8個）

- 機能名：契約書アップロード/条文抽出
  - できること：`.docx/.pdf` からタイトル/前文/条項を抽出
  - 利用者メリット：手作業の分解が不要
  - 入力項目（必須）：契約書ファイル
  - 添付：`.docx/.pdf`
  - 制約：抽出完了まで待機

- 機能名：ナレッジ照合/LLM審査
  - できること：条項とナレッジを照合し懸念点を提示
  - 利用者メリット：見落としを減らす
  - 表示される情報：懸念有無、関連ナレッジ

- 機能名：審査結果表示
  - できること：条項ごとの審査結果を一覧化
  - 利用者メリット：懸念条項を即確認
  - 表示される情報：条項、審査結果、懸念あり自動展開

- 機能名：審査結果CSV出力
  - できること：契約基本情報+条項結果をCSVで出力
  - 利用者メリット：共有/保存が容易

- 機能名：審査チャット
  - できること：審査結果に対する追加質問
  - 利用者メリット：解釈の補助

- 機能名：ナレッジ検索/閲覧
  - できること：契約種別/キーワードで検索
  - 利用者メリット：審査観点の再利用

#### 3.2 便利機能（差別化：3〜6個）

- モデル選択（審査モデルの切替）
- 懸念あり条項の自動展開
- 未紐付けナレッジの別枠表示
- ナレッジ一覧のExcelダウンロード
- ナレッジ一覧のページネーション

#### 3.3 アカウント・セキュリティ（必要最小限で）

- ログイン方式：起動のみ（管理者操作は `KNOWLEDGE_ADMIN_PASSWORD`）
- 権限：一般利用者/管理者（ナレッジ編集のみ差分）
- データ扱い：ナレッジ/審査結果はDB管理、添付は解析用途

### 4. 画面一覧（インフォグラフィックの「絵」の元ネタ）

- ホーム/ダッシュボード：タイトル/バージョン、審査/ナレッジへのリンク
- 契約審査：アップロード、結果表示、審査チャット、CSV出力
- ナレッジ管理フォーム：検索/フィルタ、編集（管理者）
- ナレッジ一覧編集：Data Editor、Excel出力、一括更新（管理者）

### 5. よくある質問（利用者が詰まる場所だけ）

- Q：対応ファイル形式は？ → A：`.docx/.pdf`
- Q：編集できるのは誰？ → A：管理者のみ（`KNOWLEDGE_ADMIN_PASSWORD`）
- Q：緊急時は？ → A：社内運用窓口に連絡

  - ナレッジが再利用できる
